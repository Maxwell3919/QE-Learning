# Citation and Source Policy

## 1. 为什么需要这个标准？

QE 笔记如果没有来源边界，很容易把官方参数定义、教程经验、个人推断和未验证说法混在一起。本仓库要求可执行知识点都可追踪、可复查、可更新。

本标准回答三个问题：

- 什么内容可以作为来源。
- 不同类型断言应该优先核对哪里。
- 新增或修改页面时怎样检查来源是否合格。

## 2. 允许引用什么？

仓库只引用公开网页、官方文档、论文或教程链接。不引用本地私有文件路径。

允许来源包括：

- QE 官方文档、官方 guide、官方 input reference。
- 公开教程、公开课程材料、公开工具文档。
- 公开论文、预印本、期刊网页或 DOI 页面。
- 本仓库已有规范页、导读页和来源索引页。

禁止来源包括：

- 本地私有路径，例如个人电脑、集群 home、Downloads、临时 scratch、私有项目目录。
- 无公开入口的聊天记录、截图、私有笔记或他人未公开文件。
- 带 tracking 参数、社交平台点击参数、广告点击参数、会话 token 或登录态参数的 URL。
- “网上常见说法”“很多教程都这么写”“我记得”“一般都这样”等无法追踪来源的表达。

如果某个判断来自个人经验，可以写成“本页采用的学习约定”或“本页的复查建议”，但必须说明依据、适用范围和需要回到哪些公开来源核对。

## 3. 来源优先级

不同断言使用不同来源优先级：

- 参数定义、默认值、输入卡片、程序行为：优先 QE 官方 `INPUT_*` reference 或官方 package guide。
- 程序链、文件关系、运行顺序：优先官方 guide；教程只能作为学习拆解和交叉验证。
- 输出解释、warning、收敛状态：优先官方文档和实际 output 证据；教程经验必须标注适用边界。
- 工具能力和限制：优先工具官方文档；本仓库工具页只说明它在 QE workflow 中的位置，不替代工具文档。
- 理论背景、方法假设、物理解释：优先论文、教材型公开资料或官方文档中的 theory/guide 章节。

当教程、工具文档和官方文档说法不一致时，涉及 QE 输入语法和参数含义的断言以 QE 官方文档为准；涉及第三方工具用法的断言以该工具官方文档为准。

## 3.1 Canonical source tier

论文和方法来源使用 [references/canonical-literature.md](../references/canonical-literature.md) 中的分级。新增或修改页面时，应把来源类型写清楚：

| 等级 | 用途 | 可支撑的断言 |
|---|---|---|
| A0 | 基础理论或不可跳过的方法原点 | DFT/KS/DFPT 等原则边界 |
| A1 | workflow 或工具链核心方法 | 程序链、方法适用范围、核心数据依赖 |
| A2 | failure-mode 判断必读 | band gap problem、pseudo transferability、U provenance 等判断边界 |
| B1 | 公开课程、教程或综述入口 | 学习顺序、操作拆解、专题入口 |
| B2 | 本仓库学习约定和审阅模板 | `PASS / WARN / BLOCK`、记录字段、维护约定 |
| C | 未核验或临时线索 | 不得作为正式页面断言来源 |

来源等级不替代官方文档。凡是涉及 QE 字段、默认值、允许取值、文件格式或版本行为，仍必须回到 QE 官方 `INPUT_*` 或对应工具官方文档核验。

## 4. input 模板来源标注

每个 input 模板必须说明来源类型，不能只给出模板本身。

推荐写法：

- `来源类型：官方文档改写`，用于直接根据 QE 官方 reference 或 guide 整理的模板。
- `来源类型：外部教程改写`，用于从公开教程学习流程后重写的模板。
- `来源类型：自建学习示例`，用于为说明参数位置、文件链或 review 步骤而构造的最小示例。

每个模板至少说明：

- 对应 QE 程序，例如 `pw.x`、`ph.x`、`bands.x`。
- 模板意图，例如 SCF、relax、DOS、phonon 或后处理。
- 参数定义核对入口，例如 QE 官方 `INPUT_*` reference。
- 若来自教程，说明教程只提供 workflow 参考，参数含义仍需回到官方文档核对。

模板不得声称“推荐参数”“通用设置”“最佳实践”，除非页面给出公开来源、适用范围和反例边界。

## 5. 工具和教程导读来源规则

`references/tutorial-sites/` 和 `references/tools/` 的页面是导读页，不是外部资料的复刻。

教程导读页应写清：

- 该教程适合回答什么问题。
- 它适合放在学习路径的哪个阶段。
- 哪些结论必须回到 QE 官方文档核对。
- 哪些内容可能是教程作者的教学简化或特定环境选择。

工具导读页应写清：

- 工具在 QE workflow 中帮助解决什么问题。
- 工具不能替代什么，例如 QE input 理解、output review、收敛判断或物理解释。
- 使用工具时需要核对哪些原始工具文档。
- 工具页不是推荐榜，不按流行度排序。

导读页不应堆链接。正文只保留阅读顺序、适用边界和核对方式；原始链接集中放在页面末尾或 [source-index.md](../references/source-index.md)。

## 6. 版本敏感断言

QE 官方 input reference、工具 API、教程页面和第三方文档都可能随版本变化。以下内容属于版本敏感断言：

- 参数默认值、允许取值、废弃状态或新增参数。
- 程序行为、输出格式、文件名约定或重启方式。
- 第三方工具的命令、API、接口格式或支持范围。
- 依赖特定 QE 版本、工具版本、库版本或网页更新时间的教程步骤。

写版本敏感断言时必须做到：

- 标注核对来源，例如官方 reference、guide、工具文档或论文。
- 尽量标注版本、文档日期、页面版本或访问日期；如果来源页面没有版本信息，写明“页面未标注版本”。
- 避免写成永久结论；优先使用“在所引来源版本中”“本页核对时”“需按当前官方文档复查”等表述。
- 修改旧页面时重新核对版本敏感断言，不要沿用旧链接下的旧结论。

如果暂时无法核对版本，不要写成事实。可以放入“需复查断言”或页面维护备注。

## 7. 页面末尾来源

来源集中放在页面末尾，正文只在必要处用简短来源标签提示，不在段落中堆多个链接。

页面末尾建议使用：

```markdown
## Sources

- `来源名称：公开链接`：用于核对的内容。
- [Source Index](../references/source-index.md)：完整来源索引。
```

来源条目应说明“用于核对的内容”，而不是只放 URL。若页面只使用本仓库已有来源索引，可以只链接 [source-index.md](../references/source-index.md)，但正文中不能加入未在索引或页面末尾出现的外部来源。

## 8. 新增页面 source checklist

新增页面或大幅修改页面前，按以下清单检查：

- 页面中是否有参数定义、默认值、输入语法或程序行为断言；若有，是否回到 QE 官方文档核对。
- 页面中是否有教程经验、工具建议或 workflow 顺序；若有，是否说明它是教程导读、工具边界还是本仓库学习约定。
- 页面中是否出现本地私有路径、未公开文件、截图来源、聊天记录或登录态链接；若有，删除或改成公开来源。
- 页面中是否出现 tracking URL；若有，清理到 canonical URL。
- 页面中是否出现“网上常见说法”“我记得”“一般都这样”等无法追踪表达；若有，改成有来源的断言或删除。
- 页面末尾是否集中列出来源，并说明每个来源用于核对什么。
- 版本敏感断言是否标注来源版本、页面版本、访问日期或“页面未标注版本”。
- 页面是否把来源页写成链接堆；若是，合并到来源索引，只保留必要导读。

如果任一项无法通过，页面可以先保留为草稿，但不能作为可执行规范或教程入口。
